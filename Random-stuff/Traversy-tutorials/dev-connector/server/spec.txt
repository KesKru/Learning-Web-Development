Backend API endpoints                  Method  |  Access   |  Function
-----------------------------------------------------------------------------------------
users
----------

…/api/users/all                        GET     |  Public   |  Get all users
…/api/users/current                    GET     |  Private  |  Get current user

-----------------------------------------------------------------------------------------
profiles
----------

…/api/profiles                         GET     |  Private  |  Get current users profile
…/api/profiles/all                     GET     |  Public   |  Get all profiles
…/api/profiles/handle/:handle          GET     |  Public   |  Get profile by handle
…/api/profiles/user/:user_id           GET     |  Public   |  Get profile by user id

…/api/profiles                         POST    |  Private  |  Create new user profile
…/api/profiles/experiance              POST    |  Private  |  Add experiance to profile
…/api/profiles/education               POST    |  Private  |  Add education to profile

…/api/profiles/experiance/:exp_id      DELETE  |  Private  |  Delete experiance form profile
…/api/profiles/education/:edu_id       DELETE  |  Private  |  Delete education form profile
…/api/profiles/                        DELETE  |  Private  |  Delete user and profile

-----------------------------------------------------------------------------------------
posts
----------

…/api/posts                            GET     |  Public   |  Get all posts
…/api/posts/:id                        GET     |  Public   |  Get a post by id

…/api/posts                            POST    |  Private  |  Create new post
…/api/posts/like/:id                   POST    |  Private  |  Like a post
…/api/posts/unlike/:id                 POST    |  Private  |  Unlike a post
…/api/posts/comment/:id                POST    |  Private  |  Add comment to post

…/api/posts/:id                        DELETE  |  Private  |  Delete a post
…/api/posts/comment/:id/:comment_id    DELETE  |  Private  |  Delete comment to post

-----------------------------------------------------------------------------------------

Server Folder structure                Description
-----------------------------------------------------------------------------------------
server                                 -Top level server folder
├─config                                 -Configuration files folder
│  │─keys.js                               -Various keys, url, secrets, etc. in variables
│  └─passport.js                           -PassportJs configuration
├─models                                 -Folder for MongoDb models
│  │─Post.js                               -MongoDb model for posts
│  │─Profile.js                            -MongoDb model for profiles
│  └─User.js                               -MongoDb model for users
├─routes                                 -Folder for routes
│  └─api                                   -api routes folder
│     │─Post.js                               -Posts routes
│     │─Profile.js                            -Profile routes
│     └─User.js                               -Users routes
│─validation                             -Folder for input validation
│  │─education.js                          -Education input validation
│  │─experiance.js                         -Experiance input validation
│  │─isEmpty.js                            -Helper function to check for empty strings
│  │─login.js                              -Login input validation
│  │─post.js                               -Posts input validation
│  │─profile.js                            -Profiles input validation
│  └─register.js                           -Register input validation
│─.gitignore                             -Tell git what to ignore
│─package-lock.json                      -package-lock.json
│─package.json                           -Project dependancies
└─server.js                              -Main server file
-----------------------------------------------------------------------------------------
